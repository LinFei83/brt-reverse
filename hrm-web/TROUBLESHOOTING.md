# 故障排除指南

## 常见问题

### 1. 模式切换时设备需要重启

**现象**: 点击模式切换按钮后，提示"模式切换命令已发送，设备需要重启以应用新模式，请稍后重新连接设备"。

**原因**: 这是设备的正常行为。心率带在接收到模式切换命令后，会断开蓝牙连接并重启，以新的工作模式启动。

**操作流程**:
1. 点击目标模式按钮（如"心电模式"）
2. 系统发送切换命令到设备
3. 设备断开连接并重启（约5-10秒）
4. 等待设备重启完成后，点击"连接设备"按钮
5. 重新连接后，界面会显示新的工作模式

**注意**: 这不是错误，而是设备切换模式的必要过程。模式切换命令已成功发送，只需等待设备重启后重新连接即可。

### 2. 工作模式显示"未知"

**现象**: 连接设备后，工作模式栏显示"未知"。

**原因**: 设备需要一些时间来响应配置查询命令。

**解决方案**:
1. 等待3-5秒，系统会自动读取设备模式
2. 如果仍显示"未知"，尝试断开并重新连接设备
3. 检查后端日志，确认是否收到了 SportModel/SR/XO 响应

### 3. 图表不显示数据

**现象**: 设备已连接，但图表没有数据。

**可能原因**:
- 当前模式下不产生该类型的数据
  - Sport 模式: 主要是加速度和基础心率
  - ECG 模式: 高精度心电图数据
  - HRV 模式: R-R间期数据
- WebSocket 连接未建立

**解决方案**:
1. 检查浏览器控制台是否有 WebSocket 连接错误
2. 确认选择了正确的模式
3. 刷新页面重新建立连接

### 4. 无法连接设备

**现象**: 点击"连接设备"后，提示连接失败。

**可能原因**:
- 设备地址配置错误
- 设备未开机或电量不足
- 蓝牙适配器未启用
- 设备已被其他程序连接

**解决方案**:
1. 检查 `backend/config.py` 中的设备地址是否正确
2. 确认设备已开机且电量充足
3. 检查系统蓝牙是否启用
4. 关闭其他可能连接该设备的程序
5. 重启设备和蓝牙适配器

### 5. 端口被占用

**现象**: 启动服务时提示端口 8000 被占用。

**解决方案**:
1. 修改 `backend/config.py` 中的 `PORT` 配置
2. 或者关闭占用端口的其他程序

## 调试技巧

### 查看后端日志

后端会输出详细的日志信息，包括：
- 设备连接状态
- 接收到的数据包类型
- 模式切换过程
- 错误信息

### 查看浏览器控制台

按 F12 打开开发者工具，查看：
- WebSocket 连接状态
- 接收到的实时数据
- JavaScript 错误

### 测试 API 端点

使用 curl 或浏览器直接访问：
- `http://localhost:8000/api/status` - 查看设备状态
- `http://localhost:8000/docs` - FastAPI 自动生成的 API 文档

## 最佳实践

1. **模式切换流程**:
   - 连接设备
   - 等待模式识别完成（3-5秒）
   - 点击目标模式按钮
   - 如果断开，等待5秒后重新连接
   - 确认新模式已生效

2. **数据采集**:
   - 保持设备与电脑距离在 10 米以内
   - 避免频繁切换模式
   - 长时间采集时注意设备电量

3. **性能优化**:
   - 图表默认显示最近 500 个数据点
   - 可在 `frontend/js/charts.js` 中调整 `maxDataPoints`
   - 高频数据采集时建议降低图表更新频率

## 技术支持

如遇到其他问题，请检查：
1. Python 版本 >= 3.8
2. 所有依赖已正确安装
3. 操作系统的蓝牙驱动正常工作
